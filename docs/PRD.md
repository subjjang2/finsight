# PRD: finsight

## 목표
카드 명세서 CSV를 업로드하면 Claude(Sonnet)가 거래를 자동 분류·집계해 지출 인사이트를 보여주는 AI 가계 분석 SaaS.

## 사용자
- 자기 카드 소비를 카테고리별로 빠르게 파악하고 월별 추이를 보고 싶은 개인. 엑셀로 직접 분석하긴 번거로운 사람.
- 한국어 UI 기준 국내 타겟. (CSV 파싱은 AI 매핑이라 카드사/포맷 무관)

## 핵심 기능
1. **CSV 업로드 + AI 컬럼 매핑** — 카드사마다 다른 헤더/날짜형식/인코딩을 Claude가 `date/merchant/amount`로 자동 매핑하고, 확인 단계 없이 곧바로 분석한다(`docs/UX_GUIDE.md` §3). 파싱·검증 후 Storage + DB 저장.
2. **AI 카테고리 분류 + 집계** — 고정 12개 카테고리(enum, 아래 `카테고리` 절)로 각 거래를 분류하고 카테고리별 합계·비율을 인사이트(요약 + 표)로 표시.
3. **월별 추이 / 이력 보관** — 업로드한 거래내역과 분석 결과를 저장해 월별 소비 추이를 비교.
4. **구독 결제(Polar)** — Free(월 5건·기본 분석) / Pro(무제한·심층 분석).

## 티어
| 티어 | 한도 | 분석 모델 | 결제 |
|------|------|----------|------|
| Free | 월 5건 분석 | Claude Sonnet | ₩0 |
| Pro | 무제한 (공정사용 상한 월 200건) | Claude Sonnet | Polar 구독 $9/월 (USD) |

게이팅 기준: **분석 횟수** (캘린더 월 단위 카운트, 매월 리셋).

## 카테고리 (확정 12개 enum)
`dining` 식비 · `shopping` 쇼핑 · `grocery` 마트·생필품 · `cafe` 카페·간식 · `transport` 교통 · `utilities` 주거·통신 · `leisure` 문화·여가 · `medical` 의료·건강 · `finance` 보험·금융 · `education` 교육 · `travel` 여행·숙박 · `etc` 기타

AI는 이 enum 밖의 카테고리를 생성하지 않는다. (구현 참조: `docs/prototype/js/data.jsx`의 `CATEGORIES`)

## MVP 제외 사항
- 보상 트랜잭션·고아 파일 정리 잡 (실패 시 단순 에러)
- Claude 재시도·백오프·회로차단 (1회 호출, 실패 시 503)
- 한도 경계 동시성 원자화 (단순 카운트 조회)
- 서술형 절약 제안·이상거래 탐지 (추후 유료 차별화로 검토)
- PII 마스킹 추가 레이어 (Supabase 기본 암호화 + RLS로 충분)
- 다중 결제수단·연간 플랜·환불 UI·쿠폰 (Polar 호스팅 페이지 위임)
- 소셜 로그인 다종·MFA (이메일/비번 1종)
- PDF/공유·차트 라이브러리 과다 (텍스트 + 간단 표)
- 백그라운드 잡/큐 (동기 처리로 충분)
- 다국어(i18n) — 한국어 단일
- 관리자 대시보드·팀/조직·역할

## 디자인
- 다크모드 고정, 도구(대시보드)처럼. 마케팅 슬롭 금지.
- 무채색(neutral) + 포인트 색 1가지(emerald). 상세는 `docs/UI_GUIDE.md`.

## 미결 사항 (구현 단계에서 확정)
1. Claude 컬럼매핑+분류를 1콜로 합칠지 2콜로 나눌지 (토큰 비용 vs 정확도). 현재 구현은 `/api/uploads`(매핑) → `/api/analyses`(분류) 2콜이며 UX는 무확인 자동 분석.
2. 동기 처리 행 상한 기본값 (Railway라 타임아웃 제약은 없음 → 비용·UX 기준)

- **고정 카테고리 12개**: 위 `카테고리` 절 (구) 미결 #2.
