# UX 가이드 — 유입·온보딩·단계 전환 설계

> 이 문서는 **사용자 흐름(UX)** 정본이다. 색/타이포/컴포넌트 같은 시각 규칙은 `docs/UI_GUIDE.md`가 정본이며 여기서 반복하지 않는다.
> 화면 목록·구현 예시는 `docs/prototype/js/screens.jsx`·`screens2.jsx`, 단계 정의는 `docs/PRD.md` 참조.
> 출처: [Figma — UI design principles](https://www.figma.com/ko-kr/resource-library/ui-design-principles/)의 7원칙을 finsight 퍼널에 특화 적용.

---

## 0. 핵심 목표 — "첫 인사이트까지의 시간(Time-to-Value)"

finsight의 단 하나의 활성화 지표는 **사용자가 첫 지출 인사이트를 보는 순간(Aha Moment)** 이다.
가입 → 업로드 → 분석 → **인사이트**. 이 4단계를 최소 마찰로 통과시키는 것이 UX의 전부다.
컬럼 매핑은 AI가 자동 처리하며 별도 확인 화면을 두지 않는다(업로드 성공 즉시 분석 시작).
모든 화면은 "지금 이 화면이 다음 단계로 넘어가는 것을 돕는가?"를 기준으로 평가한다.

비활성화는 대부분 **업로드 전(가입 직후)** 과 **분석 대기 중**에 발생한다. 이 두 지점에 설계 역량을 집중한다.

## 1. Figma 7원칙 → finsight 적용

| 원칙 | finsight 적용 |
|------|---------------|
| **계층 구조 (Hierarchy)** | 각 화면에 시선 1순위는 단 하나. 인사이트 화면은 "이번 달 총 지출 + 1위 카테고리"가 최상단. 보조 표·필터는 아래로. |
| **점진적 공개 (Progressive Disclosure)** | 가입 시 카드사·예산·목표를 묻지 않는다. CSV 1개만 받는다. 컬럼 매핑(`date/merchant/amount`)은 AI가 자동 처리해 사용자에게 묻지 않는다. Pro 기능 설명은 한도 도달 시점에만 노출. |
| **일관성 (Consistency)** | Primary 액션 버튼은 항상 화면 우하단(또는 카드 하단 우측) 고정 위치. "다음/분석/계속"의 위치·색·문구 패턴을 모든 단계에서 동일하게. |
| **대비 (Contrast)** | 각 화면의 "다음 단계로 가는 버튼"만 accent(emerald). 나머지 액션은 무채색. 파괴적 액션(삭제)은 `--down`. |
| **접근성 (Accessibility)** | 대비비 WCAG AA. 버튼 키보드 포커스/Enter 제출. 금액·상태를 색만으로 구분하지 않고 라벨·아이콘 병기. |
| **근접성 (Proximity)** | 업로드 카드 안에 "지원 형식·예시"를 함께 둔다. 로딩·결과 등 관련 정보는 한 카드 안에 묶는다. |
| **정렬 (Alignment)** | 좌측 정렬 기본, 금액은 우측 정렬(`tabular-nums`). 단계 progress는 항상 같은 그리드 위치. |

## 2. 사용자 퍼널 (단계별 전환 설계)

각 단계는 **목표(이 단계의 단 하나의 행동) / 다음 단계 트리거 / 마찰 제거 / 이탈 방지** 로 설계한다.

### 단계 1 — 랜딩 → 가입 (Acquisition)
- **목표:** "내 카드 내역, 1분 만에 분석" 가치를 한 문장으로 전달하고 가입 1개 행동만 남긴다.
- **다음 트리거:** 이메일/비번 1종. 소셜·약관 군더더기 없음.
- **마찰 제거:** 가입 폼은 필드 2개(이메일·비번)뿐. 가입 즉시 업로드 화면으로 직행(빈 대시보드를 먼저 보여주지 않는다).
- **이탈 방지:** "샘플 CSV로 먼저 보기" 보조 경로 — 가입 없이도 결과 화면을 한 번 체험하게 해 가치 증명 후 가입 유도.

### 단계 2 — 업로드 (Activation 시작)
- **목표:** CSV 파일 1개를 받는다.
- **다음 트리거:** 드래그&드롭 또는 파일 선택 → 자동으로 분석 시작(추가 클릭 없이).
- **마찰 제거:** "카드사 무관, 헤더 자동 인식"을 카드 안에 명시해 "내 카드사 되나?" 불안을 선제 해소. 지원 예시 1줄 표기.
- **이탈 방지:** 빈 상태에 **샘플 CSV 다운로드** 제공. 잘못된 파일은 에러 4상태(`UI_GUIDE.md`)로 즉시 안내하되 재시도를 같은 화면에서.

### 단계 3 — AI 컬럼 매핑 (자동, 무확인)
- **목표:** AI가 `date/merchant/amount` 컬럼을 추정하고, **확인 단계 없이** 곧바로 분석으로 넘어간다. 마찰 최소화가 우선.
- **다음 트리거:** 사용자 행동 없음. 업로드 성공 즉시 분석이 자동 시작된다.
- **마찰 제거(핵심):** 별도 매핑 확인 화면을 두지 않는다. 업로드·매핑 추정 중에는 로딩 표시("CSV를 읽고 컬럼을 매핑하는 중")로 "멈춘 게 아님"을 전달한다.
- **이탈 방지:** AI가 필수 필드를 못 잡아 분석이 실패하면(거래 0건 등) 에러를 보여주고 같은 자리에서 다시 업로드하게 한다. 이때 분석 크레딧은 소모되지 않는다(`/api/analyses`가 크레딧 차감 전에 거른다).

### 단계 4 — 분석 중 (대기 이탈 방지)
- **목표:** Claude 분석 동기 처리 대기 시간 동안 이탈을 막는다.
- **다음 트리거:** 완료 시 자동으로 인사이트 화면 전환.
- **마찰 제거:** 진행 단계 텍스트("거래 분류 중… 집계 중…")로 "멈춘 게 아님"을 전달. 스켈레톤/로딩 4상태 적용.
- **이탈 방지:** 실패 시(503) 같은 화면에서 "다시 시도" 1버튼. 사용자를 처음으로 되돌리지 않는다.

### 단계 5 — 인사이트 (Aha Moment) ★
- **목표:** "내 돈이 어디 갔는지" 3초 안에 이해시킨다. **여기가 활성화 완료 지점.**
- **계층:** 총 지출 → 1위 카테고리 → 카테고리별 표/도넛 순. 한눈에 들어오는 1개 핵심 숫자 우선.
- **다음 트리거(Retention 씨앗):** "월별 추이 보기" / "다른 명세서 추가 업로드" 2개 경로를 명확히 제시해 재방문 습관을 만든다.
- **전환 씨앗:** 무료 분석을 다 쓰지 않았어도 추이·심층 분석 같은 Pro 가치를 **자연스럽게 미리보기**(잠금 아이콘+흐림)로 노출.

### 단계 6 — 월별 추이 (Retention)
- **목표:** 2개월 이상 데이터가 쌓이면 비교 가치를 보여 재방문 이유를 만든다.
- **점진적 공개:** 데이터 1개월뿐이면 추이 대신 "다음 달에 비교가 나타납니다 + 지금 다른 명세서 추가" CTA로 빈 상태를 동기로 전환.

### 단계 7 — 무료 한도 도달 → Pro 전환 (Monetization)
- **목표:** 가치를 충분히 경험한(=한도까지 쓴) 사용자에게만 결제를 제안한다.
- **다음 트리거:** 월 5건 도달 시점에 **맥락 안에서** 업그레이드 안내(분석 시도 순간). 가입 직후 결제 강요 금지.
- **마찰 제거:** Free/Pro 차이를 표 1개로 명확히(`docs/prototype/js/data.jsx`의 `PRICING`). 결제는 Polar 호스팅 페이지로 위임.
- **정직성:** "Powered by AI" 류 과장 배지 금지(UI_GUIDE 안티패턴). Pro 가치는 실제 기능(무제한·심층 분석)으로만 설명.

## 3. 단계 전환 공통 규칙

1. **한 화면 = 한 주요 행동.** 각 화면의 Primary 버튼은 1개. 다음 단계로 가는 그 버튼만 accent.
2. **자동 전진.** 사용자 입력이 끝나면(파일 선택, 분석 완료) 가능한 한 자동으로 다음 단계로 넘긴다. 불필요한 "계속" 클릭 제거.
3. **항상 현재 위치를 안다.** 업로드 → 분석 → 인사이트의 진행 표시를 일관 위치에 둔다(점진적 공개 + 일관성).
4. **되돌아가도 잃지 않는다.** 분석 실패 시 처음으로 되돌리지 않고 업로드 자리에서 바로 재시도하게 한다.
5. **빈 상태는 막다른 길이 아니라 다음 행동.** 모든 empty 상태에 "지금 할 수 있는 1개 행동"(샘플 다운로드, 첫 업로드, 명세서 추가)을 둔다.
6. **결제는 가치 경험 뒤에.** 활성화(첫 인사이트) → 재방문(추이) → 한도 도달, 이 순서를 지키기 전에는 결제를 앞세우지 않는다.

## 4. 마이크로카피 원칙

- 동사로 시작하는 행동 지향 버튼: "분석하기", "추이 보기", "명세서 추가". ("확인", "제출" 같은 모호한 단어 지양)
- 불안 선제 해소: 업로드 전 "카드사·형식 무관, 헤더 자동 인식".
- 에러는 비난 대신 복구 경로: "파일을 읽지 못했어요. CSV인지 확인 후 다시 올려주세요."
- 한국어 단일, 금액 ₩, 날짜 `YYYY-MM-DD` (UI_GUIDE 일관).

## 5. 측정 (퍼널 점검 지표)

각 단계 통과율을 본다. 가장 낮은 전환 구간이 다음 개선 대상이다.

| 구간 | 지표 |
|------|------|
| 가입 → 업로드 | 가입 후 첫 업로드율 |
| 업로드 → 분석 | 업로드 후 분석 자동 시작·완료율 (**최대 이탈 의심**) |
| 분석 → 인사이트 | 분석 성공·인사이트 도달율(Aha) |
| 인사이트 → 재방문 | 7일 내 2번째 업로드율 |
| 한도 도달 → Pro | 업그레이드 안내 노출 대비 전환율 |
